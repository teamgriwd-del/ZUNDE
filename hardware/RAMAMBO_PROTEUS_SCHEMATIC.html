How to Build in Proteus 8
Step-by-step: place components, wire the schematic, load firmware, run simulation

CN-01 Collar Node Setup

1

Open Proteus 8 → New Project → Name: RaMambo_CN01 → Schematic: DEFAULT → PCB: None → Firmware: None

2

Press P (Pick Component) — add all parts from the Component List tab. Double-click each component to place it.

3

Place Arduino Mega 2560 in the centre of the schematic. Leave 200px space on all sides for wiring.

4

Place DS18B20 top-right. Wire DQ → Arduino D4. Add R1 (4.7kΩ) pull-up from DQ junction → VCC.

5

Place 4× POT-HG on the left. Wire wipers to A0, A1, A2, A3. Left pin of each → VCC, Right pin → GND.

6

Place LED-GREEN bottom-right of Arduino. Wire D2 → R2 (330Ω) → LED(+) → GND.

7

Place LED-RED. Wire D3 → R3 (330Ω) → LED(+) → GND.

8

Place BUZZER. Wire D8 → BUZZER(+) → GND.

9

Place VIRTUAL TERMINAL 1. Wire VT1.RXD → Arduino TX0 (pin 1). Wire VT1.TXD → Arduino RX0 (pin 0). Set baud: 9600.

10

Place BATTERY 5V. Wire (+) → Arduino VIN. Wire (-) → GND bus. Add GND power symbols on all GND pins.

11

Compile firmware in Arduino IDE (Board: Mega 2560). Sketch → Export Compiled Binary. Get the .hex file.

12

Double-click the Mega component → Program File → browse to .hex → Clock: 16 MHz → OK.

13

Press F9 (Play). Open Virtual Terminal window. Adjust POT1 to change HR. Turn POT4 fully CW → LED2 lights up, buzzer sounds.

BS-01 Base Station Setup

1

Create second Proteus project: RaMambo_BS01

2

Install com0com (free virtual COM pair tool) on Windows. Create pair: COM3 ↔ COM4.

3

Place second Arduino Mega 2560 as the BS-01 ESP32 proxy.

4

Place COMPIM component (COM Port Interface Module). Set COM port to COM4, baud 9600. Wire COMPIM.RXD → Arduino TX1 (pin 18). Wire COMPIM.TXD → Arduino RX1 (pin 19).

5

In the CN-01 project, add a second COMPIM on COM3 wired to TX1/RX1. Now the two Proteus projects communicate via virtual serial, simulating LoRa.

6

Place 4× LEDs: LED-GREEN (D2 PWR), LED-BLUE (D4 LoRa), LED-YELLOW (D13 WiFi), LED-RED (D15 Alert). Each with 330Ω → GND.

7

Place VIRTUAL TERMINAL 2 for debug output → wire to Arduino TX0/RX0 (115200 baud).

8

OLED SSD1306 is not in Proteus standard library. Replace with Virtual Terminal 3 on I2C bus, or use the formatted text output in VT2 to see OLED content.

9

Compile BS-01 firmware (Mega 2560 variant). Load .hex into BS-01 Mega component.

10

Run both Proteus projects simultaneously. CN-01 sends JSON → COMPIM → BS-01 receives → VT2 shows "[LoRa] RX ... bytes".

SSD1306 OLED in Proteus: The SSD1306 is not available in Proteus standard or advanced libraries. Use Virtual Terminal as OLED proxy — the BS-01 firmware's Serial.println statements produce equivalent output. Alternatively, connect a real SSD1306 to the serial port and use COMPIM to bridge it.
Tip: To test the full chain without com0com, open only the CN-01 project and read the JSON from Virtual Terminal 1. Copy a JSON packet and manually paste it into the BS-01 VT to simulate receiving a collar packet.

Wire Colour Convention

  • Red — +VCC / +5V / +3.3V power rail
  • Dark — GND / Ground bus
  • Green — GPIO output to LED (status)
  • Bright Red — GPIO output to LED (alert) / Buzzer
  • Yellow/Amber — 1-Wire DQ (DS18B20) / I2C
  • Purple — Analog in A0 (heart rate POT)
  • Cyan — Analog in A1, A2 (accelerometer POTs)
  • Orange — Analog in A3 (GPS drift POT)
  • Blue — UART TX/RX → Virtual Terminal
  • Cyan dashed — UART TX1/RX1 → COMPIM (LoRa bridge)

Finding components in Proteus 8:
Press P → type search term → double-click component → click to place on schematic.
COMPIM is under Virtual Instruments (same panel as Virtual Terminal).
GROUND and VCC power symbols: use Place → Power menu or press G.
To connect wires: hover near a pin until the green junction appears, then click and drag.
